# Impeccable Design Context

## Design Context

### Users
Paper traders and learners using Gainster as a sandbox to explore stock trading without real money. The interface should feel educational and explorative — empowering users to experiment, understand their decisions, and learn from outcomes. Not targeting Wall Street pros or complete beginners; the sweet spot is curious, technically-inclined people who want to understand markets through doing.

### Brand Personality
**Sharp. Minimal. Intentional.**

The interface should feel like a well-crafted tool — clean lines, restrained palette, and data that speaks for itself. Inspired by the shadcn finance dashboard showcases: modern component design with excellent typography and clear hierarchy. No gamification, no hype, no unnecessary decoration.

### Aesthetic Direction
- **Theme**: Dark mode primary, light mode as secondary
- **Color**: Monochromatic foundation (grays in OKLCH) with semantic green/red for financial indicators. Chart colors use a cool purple-blue spectrum. Avoid neon, gradients-for-the-sake-of-gradients, or excessive accent colors
- **Typography**: Geist Sans for UI, Geist Mono for numeric/financial data. Tabular nums everywhere prices and quantities appear
- **Spacing**: Breathable but not wasteful — data-dense without feeling cramped
- **Components**: shadcn/ui foundation with Base UI primitives. Composable, variant-driven (CVA)
- **References**: shadcn finance dashboard showcases, Linear-style restraint
- **Anti-references**: Gamified trading apps (Robinhood confetti), overwhelming terminals (Bloomberg density), generic SaaS templates, crypto-bro neon aesthetics

### Design Principles

1. **Data first** — Every pixel should serve the data. Charts, numbers, and tables are the product; chrome is secondary. Use hierarchy (size, weight, color) to guide the eye to what matters.

2. **Quiet confidence** — The interface should feel assured without being loud. Subtle transitions over flashy animations. Muted backgrounds with purposeful contrast on key elements. Let restraint signal quality.

3. **Learn by seeing** — As a paper trading sandbox, make outcomes legible. P&L coloring, equity curves, and trade history should tell a clear story. The UI teaches by making financial data intuitive to read.

4. **Consistent rhythm** — Uniform spacing scale, predictable component behavior, and a cohesive token system. Every card, table, and chart should feel like it belongs to the same family.

5. **Dark-native** — Design for dark mode first. Ensure contrast, readability, and depth work in dark contexts. Light mode should be a well-adapted alternative, not an afterthought, but dark is home.
